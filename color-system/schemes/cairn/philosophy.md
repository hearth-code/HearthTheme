# CairnCode Philosophy

CairnCode is a stone-neutral color philosophy for code interfaces and reading surfaces.

It treats color as a semantic language before it becomes a platform theme.
The goal is not to make every product look identical.
The goal is to let users feel the same grounded atmosphere, hierarchy, and emphasis priorities across VS Code, Obsidian, and the website.

## Core Thesis

CairnCode should feel:

- grounded but not heavy
- structured but not cold
- readable for long sessions
- stable across dark, soft, and light climates

## Vocabulary

The palette speaks through a small set of named families:

- `clay`: control-flow pressure and release accents
- `slate`: a bounded cool anchor for structure
- `lichen`: muted member and property structure
- `sand`: dry classification and type signals
- `bone`: readable carrier neutrals

## Role Philosophy

Colors are assigned by meaning before widget.

- `keyword` should feel deliberate but never loud
- `function` should anchor structure without turning the system icy
- `string` should support reading rhythm
- `comment` should guide quietly
- surfaces should reduce glare before they express personality

## Variant Philosophy

The four variants share one hierarchy and one personality.
What changes is the climate:

- `dark`: mineral-weighted default
- `darkSoft`: quieter night atmosphere
- `light`: dry-paper daylight
- `lightSoft`: calmer long-form reading light

Soft variants are not lower-contrast copies.
They are distinct climates with the same semantic contract.

## Product Principle

CairnCode exists here as a framework-validation scheme.
It proves the pipeline can host more than one color philosophy without rewriting generators, audits, or downstream delivery steps.
